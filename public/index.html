REST + SOAP API Guide • Base URL: https://rover.extio.io

Overview ▼

This learning lab helps you explore REST and SOAP APIs with real working endpoints. You’ll learn authentication, headers, parameters, and status codes using Postman.

Base URL: https://rover.extio.io
SOAP WSDL: https://rover.extio.io/soap/bookservice?wsdl

Authentication (JWT) ▼

Use your credentials to obtain a JSON Web Token (JWT). This token is required for all REST endpoints.

POST https://rover.extio.io/api/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "password"
}

Response:

{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6..."
}
Include the token in request headers:
Authorization: Bearer <token>

REST API ▼

The REST API communicates via JSON and standard HTTP verbs.

MethodEndpointDescription
GET/api/booksList all books
GET/api/books/:idRetrieve a book by ID
POST/api/booksCreate a new book
DELETE/api/books/:idDelete a book

Create Book Example

POST https://rover.extio.io/api/books
Authorization: Bearer <token>
Content-Type: application/json

{
  "title": "API Mastery",
  "author": "Jane Doe",
  "year": 2025
}

Get Book Example

GET https://rover.extio.io/api/books/1
Authorization: Bearer <token>
Status Codes: 200 OK, 201 Created, 204 No Content, 400 Bad Request, 401 Unauthorized, 404 Not Found

SOAP API ▼

SOAP uses XML envelopes and a single POST endpoint. Use any SOAP client or Postman with Content-Type: text/xml.

WSDL: https://rover.extio.io/soap/bookservice?wsdl
Endpoint: https://rover.extio.io/soap/bookservice

GetBook Request

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:book="http://example.com/books">
  <soapenv:Header/>
  <soapenv:Body>
    <book:GetBook>
      <id>1</id>
    </book:GetBook>
  </soapenv:Body>
</soapenv:Envelope>

CreateBook Request

<soapenv:Envelope xmlns:soapenv="http://schemas.xmlsoap.org/soap/envelope/" xmlns:book="http://example.com/books">
  <soapenv:Header/>
  <soapenv:Body>
    <book:CreateBook>
      <title>SOAP Learning</title>
      <author>John Smith</author>
      <year>2025</year>
    </book:CreateBook>
  </soapenv:Body>
</soapenv:Envelope>

Practice Scenarios ▼

  • Try REST without a token — expect 401 Unauthorized.
  • Send malformed JSON — expect 400 Bad Request.
  • Duplicate book creation — expect 409 Conflict.
  • Wrong SOAP namespace — expect SOAP Fault.
  • Create a book via REST and fetch it via SOAP.
  • Test expired JWT — expect 403 Forbidden.
  • Try a non-existent method — expect 405 Method Not Allowed.
  • Compare REST vs SOAP response times.
